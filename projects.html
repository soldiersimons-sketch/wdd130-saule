Project 01

Healthcare AI Workflow

Problem: Healthcare data can be difficult to use because it is sensitive, messy, and easy to misread if the model output is treated like a black box.

Approach: The project is planned around a clean workflow: prepare the data, score the case, review the output, and explain the reasoning in plain language.

Tools: Python, pandas, basic model evaluation, data cleaning, documentation

Value: The value is clarity. A useful healthcare analytics tool should help people understand the result, not just produce a number.

Project 02

Market Simulation

Problem: Business decisions often depend on assumptions about demand, pricing, timing, and customer behavior. Those assumptions should be tested before they become expensive.

Approach: The simulation compares multiple possible outcomes and shows how changing one factor can affect the whole result.

Tools: Excel, statistics, scenario planning, charts, summary tables

Value: The value is better planning. A simulation lets a team discuss risk and opportunity before choosing a direction.

Project 03

Risk Model Dashboard

Problem: Risk information is often scattered across notes, spreadsheets, and separate reports. That makes it hard to know what needs attention first.

Approach: The dashboard groups risk factors, scores them by likelihood and impact, and shows which issues should be reviewed first.

Tools: Power BI or Tableau, Excel, scoring logic, data visualization

Value: The value is prioritization. A dashboard can turn a long list of concerns into a clear next step.
